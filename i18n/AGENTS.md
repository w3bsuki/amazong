# i18n/ — i18n Infrastructure Rules

These rules apply to everything under `i18n/`.

## Non‑negotiables

- next-intl is required (no ad-hoc i18n).
- Keep locale routing and config consistent with `docs/10-I18N.md`.
- Avoid runtime secrets/PII (i18n config should be deterministic).
