# Production Readiness Audit

## Goal

Define the minimum bar for a safe marketplace launch.

## Security

- Admin routes require admin role (no leaks)
- RLS policies audited for sensitive tables
- Rate limiting for auth, listings, and chat
- No secrets/PII in logs

## Payments

- Webhook idempotency verified in staging
- Refunds tested end-to-end
- Payout release logic tested for edge cases
- Dispute playbook exists (support + engineering)

## Ops

- Moderation queue exists (reports, blocked users, escalations)
- Incident response checklist (downtime, webhook failure, fraud spike)
- Monitoring dashboards and alerting configured
