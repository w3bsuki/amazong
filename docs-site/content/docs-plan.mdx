This file defines the docs architecture and the minimum command set.

## Architecture

1. `docs/**`:
   - authoring SSOT.
2. `docs-site/**`:
   - generated internal browse mirror.
   - never author directly.
3. `docs/status/LAUNCH-READINESS.yaml`:
   - machine-readable launch state.
4. `.codex/**`:
   - runtime work state (`TASKS`, `DECISIONS`, `SHIPPED`).

## Command Set (Minimal)

### Required checks for docs/agent changes

```bash
pnpm -s docs:check
```

This runs:

1. `docs:gate`
2. `docs:site:check`
3. `docs:status:check`
4. `validate:skills -- --warn-only`

### Maintenance commands

```bash
pnpm -s docs:status:build
pnpm -s skills:sync -- --prune
pnpm -s docs
```

Use maintenance commands only when needed:

- `docs:status:build` when readiness inputs changed.
- `skills:sync` when `.codex/skills/**` changed.
- `docs` when browsing docs-site locally.

## Rules

- Keep canonical docs concise, explicit, and linkable.
- Avoid duplicated policy text across docs.
- Keep generated artifacts out of git tracking (`docs-site/public/_pagefind`).
- Do not add new doc systems outside this architecture.
